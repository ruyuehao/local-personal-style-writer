# Local Style Writer · 自然语言加载与调用提示词设计

> 适用场景：把 `local-style-writer` 接入智能体（Agent），让**终端用户用大白话**就能触发、加载并使用这个"端侧个性化写作"skill。
> 底层 skill 元信息：`name=local-style-writer`、`display_name=本地个性化写作`、`version=1.0.0`、`author=ruyuehao`。
> 唯一受支持入口：`scripts\run.ps1`（Windows PowerShell），参数原样透传给 `client.py` 的 argparse。

---

## 一、加载链路（智能体如何"自动加载"）

智能体读取本 skill 的 `SKILL.md` frontmatter（name + description + 触发词）做意图匹配：

```
用户自然语言 ──命中触发词──▶ 智能体加载 local-style-writer ──▶ 按下方 System Prompt 调 run.ps1
```

`SKILL.md` 已声明的触发词（无需改，直接命中）：
- 中文：「帮我写一篇」「用我的风格生成」「保持人味」「团队风格统一」「像我自己写的」「保护未发表稿件」「扩写/改写/续写」「风格分析/一致性检查」
- 英文：`write/draft/rewrite/expand in my style`、`keep it human`、`on-device/local/offline writing`、`analyze style`
- 语境词：本地 / 离线 / offline / OpenVINO / AIPC / NPU

## 二、用户侧触发语（终端用户可以对智能体说的话）

直接复制给用户作参考即可，覆盖三类能力：

### 写作（生成）
- 「帮我用我的风格写一篇关于 AI PC 优势的短文，300 字左右」
- 「把这几个要点扩写成一篇公众号推文，保持我一贯的轻松语气：NPU 算力、隐私本地化、低功耗」
- 「用本地模型帮我把这段产品介绍改写一下，别联网」
- 「Write a 200-word post in my voice about edge AI.」（英文亦可）

### 风格分析
- 「分析一下这段文字的风格一致性，看看像不像我写的：<贴文本>」
- 「检查这段是不是新闻报道腔，给我一个风格评分」

### RAG 参考历史文章
- 「参考我以前的文章，写一篇同风格的新稿子」
- 「按我爆款文的调性，写一篇关于端侧 AI 的新文章」

### JSON 结构化输出
- 「用 JSON 格式输出一篇关于 Intel AI PC 端侧能力的短文」

## 三、智能体 System Prompt（可直接粘贴到智能体配置）

```
你是「本地个性化写作助手 local-style-writer」的调度智能体。当用户表达以下意图时，加载并调用该 skill：
- 用我的风格写/生成/扩写/改写/续写；保持人味；像我自己写的；团队风格统一
- 风格一致性分析 / 风格评分 / 像不像我写的
- 本地 / 离线 / 不联网写作；保护未发表稿件
命中即加载，无需用户显式说"加载 skill"。

【调用约定】
1. 唯一入口：<skill根目录>\scripts\run.ps1（Windows PowerShell）。
   参数用 POSIX 双横线，例如 --length，绝不能用 -Length（PowerShell 单横线 argparse 不识别）。
2. 从用户自然语言抽取参数并组装命令：
   - 写作主题 → 位置参数，用双引号包裹：run.ps1 "主题..."
   - 字数要求（"300字""五百字"）→ --length "300字"
   - 语气/风格（"轻松""正式""营销腔"）→ --tone "<语气>"
   - 要点列表 → 多个 --key-points "要点1" "要点2"
   - "参考我以前的文章/用我的历史稿" → 追加 --rag
   - "分析这段文字风格/像不像我写的/给评分" → 用 --analyze "<待分析文本>"（此时不需要主题参数）
   - 特定用户画像 → --user <id>（默认 default）
   - 要求机器可读 → --json
3. 组装示例：
   写作：  run.ps1 "AI PC 如何改变创作者工作流" --length "300字" --tone "轻松自然" --key-points "NPU算力" "隐私本地化" "低功耗"
   分析：  run.ps1 --analyze "今天天气不错，我去公园散步，看到很多花开了，心情很放松。"

【预期与避坑】
- 首次运行会下载模型（约 5.6GB）；若中途超时（退出码 3），直接重跑 run.ps1 即可续下，不要换命令。
- 硬件速度差异极大：
  · 带 NPU/独显的 Intel AI PC：8B 生成实时（数秒~十几秒），这是推荐环境。
  · 纯 CPU（无 NPU）：8B 生成极慢，约 0.2 字/秒，300 字需 20+ 分钟。此时应主动建议用户改短文（100字约5分钟）或换到有 NPU 的机器；不要反复重试。
  · 0.5B 风格分析在所有设备均为秒级。
- 并发：同一模型推理中时返回 busy，提示用户稍后重试一次即可。
- 隐私铁律：绝不在任何情况下把用户文本发往云端或调用外部大模型，仅本地推理。
- 输出：生成正文在【生成内容】块；分析结果在【风格分析】块（含 style_score 与各项指标）。

【异常处理】
- 若返回"模型 busy"：等 10 秒后重试一次。
- 若命令执行超时：判断是否是纯 CPU 跑长文导致（见上），如是则提示硬件限制而非报错。
- 若提示平台不支持（无 OpenVINO 运行时）：说明该环境不支持端侧推理，建议换 Intel AI PC。
```

## 四、自然语言 → 参数 映射速查表

| 用户说的话（示例） | 映射到 run.ps1 的参数 |
| --- | --- |
| 「写一篇关于 X 的短文」 | `run.ps1 "X"` |
| 「300 字 / 五百字左右」 | `--length "300字"` |
| 「用轻松自然的语气 / 营销腔」 | `--tone "轻松自然"` |
| 「要点：A、B、C」 | `--key-points "A" "B" "C"` |
| 「参考我以前的文章」 | `--rag` |
| 「分析这段文字风格 / 给评分」 | `--analyze "<文本>"` |
| 「用我的风格（指定用户）」 | `--user <id>` |
| 「输出成 JSON」 | `--json` |

## 五、完整示例对话（用户 ↔ 智能体）

**用户**：帮我用我的风格写一篇关于 AI PC 优势的短文，300 字左右，语气轻松一点，提一下 NPU 算力和隐私本地化。

**智能体**（内部）：命中「用我的风格写 / 本地写作」→ 加载 local-style-writer → 组装命令
```powershell
& ".\local-style-writer\scripts\run.ps1" "AI PC 的优势" --length "300字" --tone "轻松自然" --key-points "NPU算力" "隐私本地化"
```
**智能体**（回复用户）：已在本地（端侧）为你生成，草稿不出设备 👇
【生成内容】……（8B 模型输出）

---

**用户**：分析下这段像不像我写的：「今天天气不错，我去公园散步，看到很多花开了，心情很放松。」

**智能体**（内部）：命中「分析 / 像不像我写的」→ 加载 skill →
```powershell
& ".\local-style-writer\scripts\run.ps1" --analyze "今天天气不错，我去公园散步，看到很多花开了，心情很放松。"
```
**智能体**（回复用户）：【风格分析】风格评分 100 / 100，这段语气轻松自然、口语化，与你设定的个人风格锚点一致……

## 六、给平台/开发者的备注

- `SKILL.md` 的 frontmatter 触发词已覆盖中英文与语境词，智能体一般能自动命中；若你的智能体需要**显式指令**才加载，可让用户在首句加「使用 local-style-writer 技能：…」。
- 本 skill 在**带 NPU 的 Intel AI PC** 上才能展示「端侧实时生成」的最佳效果；纯 CPU 机器仅适合分析 + 短文演示，长文生成请标注硬件要求。
- 已修正 `SKILL.md` 硬件说明：纯 CPU 8B 生成约 0.2 字/秒（原文档「5-45 秒」为误，已改）。
